Project #1: Traffic Sign Classifier (Self-driving car Nanodegree)

I utilized a homebrew convolutional neural network (CNN) to train on traffic signs to predict the input images using Python, scikit-learn, OpenCV, TensorFlow, Pandas, Numpy, matplotlib and pickle. The training data was a German traffic sign data set and the test set was U.S. traffic signs. Training achieved 98.6% accuracy by tweaking the hyperparameters and the model was able to identify the traffic signs in the test set. More details can be found in my writeup.


Project #2: Behavioral Cloning (Self-driving car Nanodegree)

I utilized a convolutional neural network (CNN) model Nvidia published and I augmented images from the original dataset to train the model using OpenCV, scipy and keras in addition to many of the same libs in Project #1 in which a car was able to drive around a previously unseen track successfully in a simulation. This was a particularly difficult challenge because my sluggish CPU and video resolution. When running the test after training, it was problematic because the model couldn't reliably get a result fast enough and the car would oscillate. Lowering the video resolution minimized the performance issue. It demonstrated how important having a good development environment is to train and test to get a successful result. Note: My reviewer did not face this problem when running my model.More details can be found in my writeup.

Project #4: A* Algorithm (Data Structures & Algorithms Nanodegree)

The implementation for A* algorithm was required for the final project in this nanodegree. This was one of the many fun ones to do. Conceptually, it is easy to understand since it combines Djikstra and Greedy Best-First-Search. My implementation is probably a little different than is typically done as I used recursion. It seemed appropriate and I reinforced many of the concepts learned in the program. The blue node on the left and the yellow node on the right are the starting and goal nodes respectively. The red nodes are the lowest step cost of the path + the cost of the heuristic function estimating the cost of the cheapest path from said node to the goal. I used Euclidean distance for both distance calculations. Once the goal was reached and no other paths are shortest, then the reconstructed path is returned. This is useful for many applications where a popular one is Google Maps. Now you know why sometimes you have to merge from the rightmost to the leftmost lane on a 4-lane highway. ;)
